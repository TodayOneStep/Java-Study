# [TIL] 절차 지향 vs 객체 지향 프로그래밍 (OOP)

<br/>

## 💡 개요 및 핵심 요약
오늘 **절차 지향(Procedural)** 과 **객체 지향(Object-Oriented)** 을 공부했다. 흔히 "절차 지향은 옛날 것, 객체 지향은 좋은 것"이라는 오해가 있지만, 둘은 우열의 관계가 아니라 **문제를 바라보는 관점(패러다임)의 차이**이다.

* **절차 지향:** 세상의 모든 일을 **'동사(순서와 흐름)'** 로 바라보고 함수로 쪼개는 방식.
* **객체 지향:** 세상의 모든 것을 **'명사(독립된 부품/객체)'** 로 바라보고 관계를 맺어주는 방식.

<br/>

## 💡 절차 지향 프로그래밍 (Procedural Programming)
* **핵심:** 프로그램이 수행해야 할 **'일의 순서(프로세스)'** 와 흐름에 초점을 맞춤.
* **특징:** 데이터(변수)와 기능(함수)이 철저하게 **분리**되어 있음. 함수가 외부 데이터를 가져와서 순차적으로 가공함.
* **장점:** 컴퓨터의 연산 구조(CPU)와 유사하여 **실행 속도가 압도적으로 빠름**.
  * 직관적이고 군더더기가 없어 소규모 프로그램이나 알고리즘 처리에 유리함.
* **단점:** 프로그램이 커지면 데이터와 함수가 복잡하게 얽혀 **유지보수가 매우 어려워짐** (스파게티 코드 위험).
* **적합한 분야:** 하드웨어 제어(임베디드), 에어백 센서 소프트웨어, 고속 그래픽/데이터 연산 등.

<br/>

## 💡 객체 지향 프로그래밍 (OOP)
* **핵심:** 상태(데이터)와 행위(기능)를 가진 **'객체(Object)'들의 상호작용**으로 프로그램을 구성함.
* **특징:** 데이터와 기능을 하나의 객체 안에 묶어서 보호함 (**캡슐화**).
* **장점:** 코드를 독립된 부품(클래스)으로 만들기 때문에 **유지보수와 확장이 매우 용이함**.
  * 대규모 프로젝트에서 여러 명이 협업하기 좋음.
* **단점:** 뇌의 자연스러운 생각 흐름(순서)과 달라 **초기 설계 비용과 진입 장벽이 높음**.
  * 객체 생성 및 호출 과정에서 절차 지향에 비해 미세한 오버헤드(메모리/속도)가 발생할 수 있음.
* **적합한 분야:** 대규모 엔터프라이즈 시스템, 웹 서비스(Spring 등), 게임 개발 등 복잡한 비즈니스 로직.

<br/>

## 💡 Plus

* **객체 지향에서도 순서는 중요하다.**
   * "절차(Procedure)"라는 단어 때문에 오해하기 쉽지만, 컴퓨터는 무조건 명령어를 순서대로 실행한다. 객체 지향 역시 객체 간의 관계를 설계할 뿐, 객체의 메서드 내부나 객체끼리 메시지를 주고받을 때는 순차적인 흐름(절차)에 따라 작동한다.
* **객체 지향이 무조건 쉽거나 정답인 것은 아니다.**
   * 인간의 뇌는 기본적으로 절차 지향(시간의 흐름)으로 생각하기 때문에, 객체 지향의 고차원적인 개념(캡슐화, 상속, 다형성)과 설계 원칙을 배우는 것은 훨씬 어렵다. 
   * 작은 프로그램이나 단순 계산은 오히려 절차 지향이 훨씬 효율적이다.

<br/>

## 💡 Java 코드로 보는 직관적 비교

### ① 절차 지향 스타일 (데이터와 기능의 분리, 순서 중심)
```java

class CarData {
    int fuel = 50;
}

class CarService {
    public static void drive(CarData data, int distance) {
        data.fuel -= distance * 0.1;
        System.out.println(distance + "km 주행. 남은 기름: " + data.fuel + "L");
    }
}

public class Main {
    public static void main(String[] args) {
        CarData myCarData = new CarData();
        CarService.drive(myCarData, 100);
    }
}
```

<br/>

### ② 객체 지향 스타일 (데이터와 기능의 결합, 객체 간의 관계)
```java
class Car {
    private int fuel; 

    public Car(int fuel) { this.fuel = fuel; }

    public void drive(int distance) { 
        this.fuel -= distance * 0.1;
        System.out.println(distance + "km 주행. 남은 기름: " + this.fuel + "L");
    }
}

public class Main {
    public static void main(String[] args) {
        Car myCar = new Car(50); 
        myCar.drive(100);       
    }
}
```
